The `typeorm init` command generates a ready-to-run project with all files configured for your chosen database and module system.

<Note>
  The CLI quickstart targets Node.js applications. If you are targeting Cordova, React Native, Expo, or NativeScript, follow the [Getting Started](/getting-started) guide instead and configure your DataSource manually.
</Note>

<Steps>
  <Step title="Scaffold the project">
    Run `npx typeorm init` with a project name and database type:

    ```bash
    npx typeorm init --name MyProject --database postgres
    ```

    The `--database` flag accepts: `postgres`, `mysql`, `mariadb`, `better-sqlite3`, `mssql`, `oracle`, `mongodb`, `cockroachdb`, or `spanner`.

    The command creates a `MyProject/` directory, writes all project files, and runs `npm install` automatically.

    <Tip>
      Run `typeorm init` without `--name` to scaffold into the current directory. Be careful — it may overwrite existing files.
    </Tip>
  </Step>

  <Step title="Review the generated project structure">
    ```text
    MyProject
    ├── src
    │   ├── entities
    │   │   └── User.ts          # Sample entity
    │   ├── migrations           # Migration files go here
    │   ├── data-source.ts       # DataSource configuration
    │   └── index.ts             # Application entry point
    ├── .gitignore
    ├── package.json
    ├── README.md
    └── tsconfig.json
    ```

    Each file has a specific role:

    | File | Purpose |
    |---|---|
    | `src/data-source.ts` | DataSource configuration — database type, host, credentials, entities |
    | `src/entities/User.ts` | Sample entity with `id`, `firstName`, `lastName`, `age` columns |
    | `src/migrations/` | Directory for migration files generated by the CLI |
    | `src/index.ts` | Initializes the DataSource and runs a sample insert + query |
    | `tsconfig.json` | TypeScript config with `emitDecoratorMetadata` and `experimentalDecorators` enabled |
  </Step>

  <Step title="Configure the DataSource">
    Open `src/data-source.ts` and update the connection settings for your database:

    ```typescript src/data-source.ts
    import "reflect-metadata"
    import { DataSource } from "typeorm"
    import { User } from "./entities/User"

    export const AppDataSource = new DataSource({
        type: "postgres",
        host: "localhost",
        port: 5432,
        username: "root",
        password: "password",
        database: "typeorm",
        synchronize: true,
        logging: false,
        entities: [User],
        migrations: [],
        subscribers: [],
    })
    ```

    At minimum, update `host`, `username`, `password`, and `database` to match your local database. The `port` default is `5432` for PostgreSQL; adjust it for other databases.

    <Warning>
      `synchronize: true` drops and recreates columns to match your entities on startup. It is safe for local development but **must not be used in production**. Switch to [migrations](/migrations/why) before deploying.
    </Warning>
  </Step>

  <Step title="Run the application">
    ```bash
    cd MyProject
    npm start
    ```

    The generated `index.ts` initializes the DataSource, inserts a sample user, and prints the results:

    ```typescript src/index.ts
    import { AppDataSource } from "./data-source"
    import { User } from "./entities/User"

    AppDataSource.initialize().then(async () => {

        console.log("Inserting a new user into the database...")
        const user = new User()
        user.firstName = "Timber"
        user.lastName = "Saw"
        user.age = 25
        await AppDataSource.manager.save(user)
        console.log("Saved a new user with id: " + user.id)

        console.log("Loading users from the database...")
        const users = await AppDataSource.manager.find(User)
        console.log("Loaded users: ", users)

    }).catch(error => console.log(error))
    ```

    If the connection succeeds and the insert completes, your project is working correctly.
  </Step>
</Steps>

## Available init flags

Customize the scaffolded project with these flags:

| Flag | Values | Default | Description |
|---|---|---|---|
| `--name`, `-n` | string | (current dir) | Name of the project directory to create |
| `--database`, `--db` | `postgres` \| `mysql` \| `mariadb` \| `better-sqlite3` \| `mssql` \| `oracle` \| `mongodb` \| `cockroachdb` \| `spanner` | `postgres` | Database driver to configure |
| `--module`, `--ms` | `commonjs` \| `esm` | `commonjs` | JavaScript module system |
| `--express` | boolean flag | false | Include an Express server with CRUD routes and a UserController |
| `--docker` | boolean flag | false | Generate a `docker-compose.yml` for the selected database |
| `--pm`, `--manager` | `npm` \| `yarn` | `npm` | Package manager used to install dependencies |

### Generate an ESM project

```bash
npx typeorm init --name MyProject --database postgres --module esm
```

The ESM template sets `"type": "module"` in `package.json` and uses `node --loader ts-node/esm` as the start script.

### Include an Express server

```bash
npx typeorm init --name MyProject --database mysql --express
```

This adds `src/routes.ts` and `src/controllers/UserController.ts` with GET, POST, and DELETE endpoints for the `User` entity, and starts an Express server on port 3000.

### Generate a Docker Compose file

```bash
npx typeorm init --name MyProject --database postgres --docker
```

This creates a `docker-compose.yml` with a pre-configured container for your database. Start it before running the app:

```bash
docker-compose up -d
npm start
```

<Note>
  Docker Compose generation is not available for SQLite (no server required) or Oracle.
</Note>
