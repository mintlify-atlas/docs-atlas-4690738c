---
title: "QueryRunner API"
description: "Complete reference for the QueryRunner schema-builder methods available inside migration up and down methods."
---

Inside `up` and `down`, you have access to a `QueryRunner` instance. In addition to raw `queryRunner.query()`, it exposes a full schema-builder API for creating and modifying tables, columns, indexes, constraints, and more.

## Example migration using the API

```typescript
import {
    MigrationInterface,
    QueryRunner,
    Table,
    TableIndex,
    TableColumn,
    TableForeignKey,
} from "typeorm"

export class CreateQuestionAndAnswer1699000000000 implements MigrationInterface {
    async up(queryRunner: QueryRunner): Promise<void> {
        await queryRunner.createTable(
            new Table({
                name: "question",
                columns: [
                    { name: "id", type: "int", isPrimary: true },
                    { name: "name", type: "varchar" },
                ],
            }),
            true,
        )

        await queryRunner.createIndex(
            "question",
            new TableIndex({
                name: "IDX_QUESTION_NAME",
                columnNames: ["name"],
            }),
        )

        await queryRunner.createTable(
            new Table({
                name: "answer",
                columns: [
                    { name: "id", type: "int", isPrimary: true },
                    { name: "name", type: "varchar" },
                    { name: "created_at", type: "timestamp", default: "now()" },
                ],
            }),
            true,
        )

        await queryRunner.addColumn(
            "answer",
            new TableColumn({ name: "questionId", type: "int" }),
        )

        await queryRunner.createForeignKey(
            "answer",
            new TableForeignKey({
                columnNames: ["questionId"],
                referencedColumnNames: ["id"],
                referencedTableName: "question",
                onDelete: "CASCADE",
            }),
        )
    }

    async down(queryRunner: QueryRunner): Promise<void> {
        const table = await queryRunner.getTable("answer")
        const foreignKey = table.foreignKeys.find(
            (fk) => fk.columnNames.indexOf("questionId") !== -1,
        )
        await queryRunner.dropForeignKey("answer", foreignKey)
        await queryRunner.dropColumn("answer", "questionId")
        await queryRunner.dropTable("answer")
        await queryRunner.dropIndex("question", "IDX_QUESTION_NAME")
        await queryRunner.dropTable("question")
    }
}
```

## Introspection

### `getDatabases()`

```typescript
getDatabases(): Promise<string[]>
```

Returns all available database names, including system databases.

---

### `getSchemas(database?)`

```typescript
getSchemas(database?: string): Promise<string[]>
```

Returns all available schema names, including system schemas. Relevant for SQL Server and PostgreSQL only.

| Parameter | Description |
|-----------|-------------|
| `database` | When provided, returns schemas within that database. |

---

### `getTable(tableName)`

```typescript
getTable(tableName: string): Promise<Table | undefined>
```

Loads a table by name from the database. Returns `undefined` if the table does not exist.

---

### `getTables(tableNames)`

```typescript
getTables(tableNames: string[]): Promise<Table[]>
```

Loads multiple tables by name.

---

### `hasDatabase(database)`

```typescript
hasDatabase(database: string): Promise<boolean>
```

Returns `true` if a database with the given name exists.

---

### `hasSchema(schema)`

```typescript
hasSchema(schema: string): Promise<boolean>
```

Returns `true` if a schema with the given name exists. SQL Server and PostgreSQL only.

---

### `hasTable(table)`

```typescript
hasTable(table: Table | string): Promise<boolean>
```

Returns `true` if the given table exists.

---

### `hasColumn(table, columnName)`

```typescript
hasColumn(table: Table | string, columnName: string): Promise<boolean>
```

Returns `true` if the specified column exists in the given table.

---

## Databases and schemas

### `createDatabase(database, ifNotExists?)`

```typescript
createDatabase(database: string, ifNotExists?: boolean): Promise<void>
```

Creates a new database. When `ifNotExists` is `true`, silently skips if it already exists.

---

### `dropDatabase(database, ifExists?)`

```typescript
dropDatabase(database: string, ifExists?: boolean): Promise<void>
```

Drops a database. When `ifExists` is `true`, silently skips if it does not exist.

---

### `createSchema(schemaPath, ifNotExists?)`

```typescript
createSchema(schemaPath: string, ifNotExists?: boolean): Promise<void>
```

Creates a new schema. For SQL Server, `schemaPath` can be `"dbName.schemaName"` to create the schema in a specific database.

---

### `dropSchema(schemaPath, ifExists?, isCascade?)`

```typescript
dropSchema(schemaPath: string, ifExists?: boolean, isCascade?: boolean): Promise<void>
```

Drops a schema. `isCascade` automatically drops all objects in the schema (PostgreSQL only).

---

## Tables

### `createTable(table, ifNotExists?, createForeignKeys?, createIndices?)`

```typescript
createTable(
    table: Table,
    ifNotExists?: boolean,
    createForeignKeys?: boolean,
    createIndices?: boolean,
): Promise<void>
```

Creates a table. Foreign keys and indices are created by default (`true`).

---

### `dropTable(table, ifExists?, dropForeignKeys?, dropIndices?)`

```typescript
dropTable(
    table: Table | string,
    ifExists?: boolean,
    dropForeignKeys?: boolean,
    dropIndices?: boolean,
): Promise<void>
```

Drops a table. Foreign keys and indices are dropped by default (`true`).

---

### `renameTable(oldTableOrName, newTableName)`

```typescript
renameTable(oldTableOrName: Table | string, newTableName: string): Promise<void>
```

Renames a table.

---

### `clearTable(tableName, options?)`

```typescript
clearTable(tableName: string, options?: { cascade?: boolean }): Promise<void>
```

Truncates all rows from a table. `cascade` clears rows in dependent tables (PostgreSQL/CockroachDB and Oracle only).

<Warning>
  `clearTable` uses `TRUNCATE`, which cannot be rolled back in most databases.
</Warning>

## In-memory SQL

These methods let you capture the SQL that would be executed without running it against the database.

### `enableSqlMemory()`

```typescript
enableSqlMemory(): void
```

Enables memory mode. Subsequent schema operations write SQL to an in-memory buffer instead of executing it.

---

### `disableSqlMemory()`

```typescript
disableSqlMemory(): void
```

Disables memory mode and flushes the buffer.

---

### `clearSqlMemory()`

```typescript
clearSqlMemory(): void
```

Flushes all memorized SQL without disabling memory mode.

---

### `getMemorySql()`

```typescript
getMemorySql(): SqlInMemory
```

Returns a `SqlInMemory` object with `upQueries` and `downQueries` arrays of resolved SQL strings.

---

### `executeMemoryUpSql()`

```typescript
executeMemoryUpSql(): Promise<void>
```

Executes all memorized up queries against the database.

---

### `executeMemoryDownSql()`

```typescript
executeMemoryDownSql(): Promise<void>
```

Executes all memorized down queries against the database.
